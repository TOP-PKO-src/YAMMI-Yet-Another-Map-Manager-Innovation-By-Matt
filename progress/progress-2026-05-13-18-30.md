## Confirmed Source Issues

1. `YetAnotherMapManager\MapData\PkoNative\PkoMapConvertor.cs:288`
   
   - export rewrites texture id `0` to fallback values `3` or `50`
   - this changes valid source data and breaks texture parity immediately

2. `YetAnotherMapManager\MapData\PkoNative\PkoMapConvertor.cs:260`
   
   - export writes offsets for every section unconditionally
   - load logic only reads sections with non-zero offsets, so source sparsity is lost and exported maps become much larger

3. Settings extraction is not safely repeatable in-process
   
   - `LoadApplicationSettings(clientPath)` re-extracts cached BMPs and can collide with existing bitmap file locks

## Reference Conclusion

- The Tales of Pirates engine source supports Yammi's general tile field order.
- The main parity problem is in Yammi's write behavior, not in its basic read layout.
